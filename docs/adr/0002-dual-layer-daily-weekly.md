# ADR-0002: 双层节奏架构--每日自动采集 + 每周 LLM 深度分析

## Status: accepted

## Context

原架构是每日一条龙走完 collect -> normalize -> score -> finalize -> report -> build -> deploy，全程规则驱动，无 LLM 介入。评分由来源类型决定而非内容质量，分类是粗糙的关键词匹配。

用户需要一个能深度理解项目质量的系统，但每天跑 LLM 分析成本和时间不可接受。

## Decision

拆分为双层节奏：
- **每日**：纯自动化，采集新项目 + 刷新追踪项目数据 + 更新可量化分 + 部署站点
- **每周一 03:00**：独立 cron，SenseNova + DeepSeek-V4-Flash，LLM 深度分析。最多 10 个子代理并行调查，结构化 JSON 输出。分析新增 + 重评全部 + 更新参照基准 + 生成报告

## Considered Options

- 每日跑 LLM 分析：成本高、耗时不可控
- 只用规则不用 LLM：评分和分类质量不够
- 每周只分析新增：旧项目评分不更新，不符合"分数动态"需求

## Consequences

- 可量化分（60分）每日更新，质量分（40分）每周更新
- 每周一是重计算日，站点内容会有一次大幅更新
- 需要独立 cron 配置和供应商管理
