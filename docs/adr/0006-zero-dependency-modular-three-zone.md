# ADR-0006: 网站保持零依赖模块化 + 三区布局

## Status: accepted

## Context

`app.js` 27 行压缩代码不可维护，618 条全量渲染无分页，报告只是裸 .md 链接。用户确认三个用户任务（发现/搜索/理解）需要在站点上各有专门区域。

## Decision

保持零依赖原则，拆分为模块化文件（js/i18n.js, data.js, filters.js, render.js, charts.js, app.js）。CSS 自定义属性替代硬编码。

三区布局：
- 首页精选区（发现）：本周新发现高质量项目
- 工具概览区（理解）：每个工具的生态规模、资源类型分布
- 多选筛选表格（搜索）：标签按钮组多选 + 6 种排序 + 分页

报告系统 3 类（生态周报/工具对比/推荐榜），每周 LLM 生成，站内渲染。

## Considered Options

- 引入前端框架：违反零依赖偏好
- ES modules：`file://` 协议不支持，本地调试不便
- 多 `<script>` 标签 + 全局变量：最简单，兼容性最好

## Consequences

- 全局变量共享需要命名约定
- 逻辑已按职责拆分，未来迁移到框架成本可接受
