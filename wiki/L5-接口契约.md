# L5-接口契约 — 数据怎么流

> 前后端协作的"合同"。前后端 Agent 写代码时以此为准，不一致就是 bug。

## 前后端通信方式

纯静态站点模式。前端不调用任何后端 API，所有数据预先构建为 JSON 文件放在 site/data/ 目录下。

前端通过 `fetch('data/<filename>.json')` 加载。

## 数据文件清单

| 文件 | 内容 | 前端使用 |
|------|------|---------|
| data/projects.json | 全量项目（2026-07-14 bulk 后 **5165**） | 主数据源（精简字段；含 **llm_summary**） |
| data/search-index.json | 轻量搜索索引 `[{id,text}]`，text 仅 name/summary/resource_type/target_tools | 搜索 includes 匹配（`SIC_data.searchMap`） |
| data/detail-index.json | `{ project_id: chunk_index }` | loadDetail 定位分片 |
| data/detail/{i}.json | 详情分片，**100 条/片**（当前 52 片） | 详情面板按需加载并缓存分片 |
| data/curated-projects.json | 推荐集（当前 **40**） | curatedOnly 筛选 |
| data/rejected-projects.json | 噪声集（当前 **10**） | 仅在 index 展示 |
| data/tools.json | 目标工具定义（~10–11） | 工具筛选下拉框 |
| data/metrics.json | 统计摘要 | 页头度量卡片 |
| data/concepts.json | 概念定义 | 当前前端未使用 |
| data/i18n.json | 分类翻译 | 当前前端未使用 |

> **已删除（批次 2）：** `data/projects-detail.json` 单体文件。构建与部署产物均不再保留。

## 数据生成源

所有 site/data/*.json 由 `scripts/build_site.py` 从以下源生成：
1. data/projects.yaml → projects.json + metrics + **search-index.json** + **detail-index.json** + **detail/{i}.json**
2. data/curated-projects.yaml → curated-projects.json
3. data/rejected-projects.yaml → rejected-projects.json
4. data/seed-tools.yaml → tools.json
5. data/concepts.yaml → concepts.json


## Rejected 项目额外字段

相比 Project 对象增加：
```
"rejection_reason": "Auto-rejected by scoring/source-quality rules: low confidence..."
```

## 数据流对照

| 前端操作 | 数据来源 | 渲染内容 |
|---------|---------|---------|
| 页面加载 → 统计卡片 | metrics.json | 总记录、推荐、拒绝数等 |
| 页面加载 → 官方工具卡片 | projects.json (ranking_scope=official) | 10 个官方工具卡片 |
| 搜索/筛选/排序 | projects.json + search-index.json（searchMap） | 项目列表表格 |
| 只看推荐 | projects.json（id 交集） | 仅 curated 项目 |
| 双语切换 | 项目 i18n + 中文模式 `llm_summary`（卡片/表格） | 字段名和摘要中英文；详情仍同时展示原生 summary 与 LLM Summary |
| 点击详情 | detail-index.json → detail/{chunk}.json | 评分明细 + quality_detail + llm_summary 等 |
| 点击报告链接 | site/reports/*.md（居中 modal） | 报告栏 pill 打开 `#reportModal` |
