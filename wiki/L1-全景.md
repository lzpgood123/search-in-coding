# L1-全景 — 项目是什么

> 所有人第一站。读完知道"这个项目是什么，它做什么，核心流程怎么走"。

## 产品定位

Search in Coding 是一个持续自动更新的 AI Coding Agent 生态追踪索引库。它不是普通 awesome list，而是一个**全自动、零人工干预的生态追踪器**，每天自动发现新资料、归一化、评分、分类、维护推荐集与噪声集，并通过正式站点展示当前生态状态。

GitHub 是完整数据与历史总仓库，正式站点是当前构建结果的公开展示面。

## 核心流程

1. 终端用户访问 https://coding.lzpgood.online/ 浏览 AI Coding Agent 生态数据
2. 通过搜索、筛选、排序找到感兴趣的工具、插件、MCP 服务器、skills 等资源
3. 点击项目链接跳转到对应的 GitHub 仓库或文章
4. 维护者（用户本人）通过 Hermes cron 每日自动运行更新，无需人工干预
5. 开发者可 Fork 仓库，替换 seed-tools 和 queries 后作为其他生态追踪器使用

## 用户视角

- **终端用户（开发者）**：通过站点浏览最新的 AI Coding 工具生态资源，按工具分类或类别查找
- **生态观察者**：通过自动生成的报告了解趋势、机会和生态对比
- **维护者**：Hermes cron 每日自动运行，偶尔手动触发立即更新或调整方向
- **复用者**：Fork 仓库，替换 data/seed-tools.yaml 和 data/queries.yaml 后追踪其他生态

## 核心功能

| 功能 | 说明 |
|------|------|
| 自动采集 | **仅 GitHub**：每日增量刷新 + 一次性历史 bulk（`initial_collection.py`，stars 分层 L1–L5b，2024-01 起）；Exa/fallback-web 已停用 |
| 数据归一化 | 原始数据合并为统一的 `data/projects.yaml` 格式 |
| 自动评分 | **100 分制**：60 分可量化（每日）+ 40 分 LLM 质量（每周）；未分析项目前端按 /60 展示 |
| 自动推荐 | auto-curated / auto-rejected 规则驱动（当前 curated **40**、rejected **10**，随评分变化） |
| 报告生成 | 自动生成 curated-top / tool-comparison / weekly-report 等报告 |
| 静态站点 | 中英文双语 SPA，零依赖纯前端，部署到 Nginx 或 GitHub Pages |
| 公共 JSON API | 公开 JSON 数据端点供机器消费 |
| 自动版本 | 日期版本号 YYYY.MM.DD，自动 CHANGELOG 和 Release |
